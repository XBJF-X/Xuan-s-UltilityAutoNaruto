# XUAN（Xuan's UltilityAutoNaruto）

### [火影忍者自动化日常工具](https://github.com/XBJF-X/Xuan-s-UltilityAutoNaruto)

## 项目原理

基于**图像检测**识别信息+最小堆调度器自动调度任务
**（不涉及任何读取和修改游戏内存数据！）**

### 参考项目：

- [Alas（前端样式）](https://github.com/LmeSzinc/AzurLaneAutoScript)
- [Mower-ng（任务执行流程和场景跳转）](https://git.zhaozuohong.vip/mower-ng/mower-ng)
- [DroidCastRaw](https://github.com/Torther/DroidCast_raw)
- [EmulatorExtras](https://github.com/MaaXYZ/EmulatorExtras)
- [MiniDevice](https://github.com/NakanoSanku/minidevice)

## 特点

- **支持主流模拟器分辨率**
  - 2560x1440
  - 1920x1080
  - **1600x900（开发所用分辨率，准确度最好）**
  - 1280x720
- **可视化自定义键位**
- **可视化选择模拟器端口（串口）**

## *II.配置*

#### Q0：模拟器需要配置哪些去配合程序？

> 分辨率设置成 `1600x900(推荐)`或者 `1920x1080` **必须是16:9比例**
> 关闭 `后台保活`
> 不要设置过于极限的性能配置（最好 `2核心2048MB`以上）

<a id="ii-q1"></a>

#### Q1：控制模式怎么选择

> 优先选择 `MiniTouch`，如果异常可以使用 `U2`

<a id="ii-q2"></a>

#### Q2：串口号怎么填写

> 首先，可以使用设置的串口选择，会自动列出可用的adb设备注意：MuMu模拟器的设备使用 `adb devices`无法列出，需要自己填写，操作如下：  
>
>> 打开你安装火影忍者的那个模拟器，点进设置的设备诊断，往下翻可以找到端口号，格式为 `5xxx 16xxx`  
>> 左侧5xxx代表串口号 `emulator-5xxx`  
>> 右侧16xxx代表串口号 `127.0.0.1:16xxx`  
>>
>
> 其中xxx是动态分配的，请按照自己的情况填写，最好使用 `127.0.0.1:16xxx`的格式填写  
> 此为模拟器通用格式，且MuMu使用emulator格式会无法连接

#### Q2.1：腾讯手游模拟器怎么连接？

> 目前不支持，该模拟器似乎没有提供ADB调试的方法

#### Q2.2：雷电火影专版怎么连接？

> 控制模式依旧 `U2`，截图模式选择 `DroidCast`或者 `U2(推荐)`，串口填写和普通版一样

#### Q3：模拟器实例索引怎么填写

> 按照模拟器多开中给出的顺序填写，比如MuMu的模拟器索引在模拟器列表缩略图的左上角 `#0`，`#1`，`#2`就是它的实例索引  
> 雷电模拟器也在多开的模拟器列表中模拟器信息的最左侧一列可以找到

#### Q4：使用[MuMu/雷电]截图方案的情况下模拟器路径该怎么填写

> MuMu模拟器，填写最顶层的安装目录即可  
> 如 `D:\Program Files (x86)\MuMuPlayer-12.0`  
> **不需要**填写子目录，交给程序自动处理即可 ！！！类似下面的：  
> `D:\Program Files (x86)\MuMuPlayer-12.0\nx_device\12.0`  
> `D:\Program Files (x86)\MuMuPlayer-12.0\nx_main`  
> 雷电模拟器填写类似 `D:/Program Files (x86)/leidian/LDPlayer9`的路径即可  

#### Q5：配置键位时程序卡死崩溃

> 报错信息 `AttributeError: 'ADB' object has no attribute 'screen_size`
>
>> 请仔细阅读[II.Q1](#ii-q1)，使用 `MiniTouch`方案
>>
>
> 报错信息和截图实例相关
>
>> 请参照[II.Q2](#ii-q2)检查截图方案
>>

## *III.使用*

#### Q1：我有多个账号，换了账号之后启动调度器为什么不执行任务了？

> 每个配置对应一个账号，进度和配置之间互相独立，有**几个账号就创建几个配置**    
> 新建配置可以通过点击左下角 `+`号创建  
> 需要做哪个号日常就执行哪个配置即可，即以账号数分配配置而不是以模拟器分配  
> 支持同时执行多个配置  

<a id="iii-q2"></a>

#### Q2：每日周胜选择什么忍者？

> 最好选择3C，这样可以做到一把过3个每日宝箱  
>
>> 忍者推荐 秽土多由也 小樱 佐助  
>> 秘卷推荐 活 霸体 散  
>> 通灵推荐 忍龟 蛤蟆健 守鹤/蛞蝓  
>>

#### Q3：秘境探险选择什么忍者？

> 须佐鼬原皮最合适，其次可以选择任何低浮空的灌伤忍者  
> 比如老佩恩，百战佩恩，老三代等等

#### Q4：为什么我设置体力/铜币购买次数之后，自己买了几次，但程序还是按照设定数买导致买多了？

> 体力和铜币购买次数没有做进度识别，也就是说只会按照设定次数购买  
> 请不要自己事先买了又执行购买，只会浪费金币

#### Q5: 为什么总览面板中日志显示 `请检查二级密码！`

> 说明该任务需要消耗金币，而你没有配置二级密码或者二级密码错误/长度不足6位  

#### Q6：为什么情报站任务卡死，情报站界面显示空白

> 这是魔方的历史遗留问题，情报站时不时就会发生卡死或加载空白的问题  
> 该任务已经重写过多次，还是没办法做到一定承受魔方的投入，见谅！  

#### Q7：为什么程序在主场景一直左右滑动，不进入正在执行的任务场景

> 请检查站街/场景，过于庞大的站街有可能遮挡场景中任务场景的入口  
> 部分开发者没有的场景也没法得到充分测试，请切换场景重试，如果依旧有问题请参照下面所述反馈  

#### Q8：为什么程序不领取组织里的三个祈福宝箱

> 因为组织里做日常的时间不统一，所以宝箱会留到第二天  
> 通过昨日奖励的方式领取，这样保证每次领取到的都是最大化的收益，避免任务的二次执行  

#### Q9：为什么清空执行时间回车之后，执行时间变空白并且没有立即执行

> 立即执行需要在**启动调度器**的情况下使用  
> （`立即执行`功能现已移动至`总览面板`下点击任务右侧`执行`按钮）  

#### Q9.1：为什么启动调度器后立即执行还是没有立即执行

> 说明立即执行不符合该任务的约束条件，某些任务是无法立即执行的  
> 比如 `天地战场`/`本服要塞战`，本身只有到**固定时间**才开放进入  
> 回车立即执行本身就是开发阶段方便测试的功能，只是未在用户端分离以备特殊情况！  
> **不是Bug，请不要再反馈给我了！！！**

#### Q10：报错显示 `127.0.0.1:16xxx/emulator-555x not online`

> 检查串口号配置和模拟器是否一致，可以参考上方[II.Q2](#ii-q2)  
> 如果一致，可以先尝试重启模拟器，可能是因为串口被占用等问题

#### Q2：怎么接收更新和bug修复？能不能自动更新？
